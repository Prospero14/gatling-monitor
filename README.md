# Gatling Monitor

Мониторинг Gatling-генераторов через SSH.

## Главная идея

**Общий кэш статусов** — любой может открыть страницу и нажать «Обновить статусы».
Результат сохраняется в памяти сервиса и **виден всем** до следующего обновления.

## SSH-креды

Пароль **не хранится** в конфиге и **не передаётся в открытом виде**:

1. Нажимаете «Обновить статусы»
2. Вводите логин/пароль в попапе
3. Браузер **шифрует** их RSA-ключом сервера
4. В DevTools в теле запроса виден только `payload` (зашифрованная строка), не пароль

После успешного обновления креды сохраняются **в памяти сервера** для авто-обновления раз в 2 часа.

## Конфигурация

`application.yml` — только генераторы и SSH-порт:

```yaml
monitor:
  ssh:
    port: 22
  generators:
    - name: generator-01
      host: 10.0.0.1
```

## Сборка и запуск

```bash
mvn clean package -DskipTests
java -jar target/gatling-monitor-1.0.0.jar
```

http://localhost:8088

## UI

- **Обновить статусы** — попап с SSH-кредами (шифрование)
- **Обновлять раз в 2 часа** — по последним успешным кредам

## API

- `GET /api/auth/public-key` — публичный RSA-ключ
- `POST /api/refresh` — тело `{ "payload": "<encrypted>" }`
- `GET /api/status` — статус и прогресс
