# Gatling Monitor

Мониторинг Gatling-генераторов через SSH.

## Главная идея

**Общий кэш статусов** — любой пользователь может зайти и нажать «Обновить статусы».
Результат сохраняется в памяти сервиса и **виден всем** до следующего ручного обновления.
При открытии страницы статусы **не пересчитываются** — показывается последний сохранённый снимок.

## Попап при неверных кредах

При неудачном входе или если при обновлении креды не подошли ни к одному генератору:

> эх, ты, простофиля, креды проверь!

Старые статусы при этом **не затираются**.

## Авто-обновление раз в 2 часа

Рубильник на главной странице — **«Обновлять раз в 2 часа»**.

- Включили → сервис каждые 2 часа обновляет статусы по **последним кредам успешного обновления**
- Первое ручное обновление сохраняет креды для авто-режима
- Если другой пользователь зашёл с **новыми** кредами и нажал **«Обновить статусы»** — отрабатывает обычная ручная логика, новые креды сохраняются для следующих авто-обновлений
- Ручное обновление всегда использует креды **текущей сессии**

## Блокировка входа

- 2 неудачные попытки → блокировка на **1 минуту** (по IP).
- После успешного входа счётчик сбрасывается.

## Запуск

```bash
cd E:\gatling-monitor
mvn spring-boot:run
```

http://localhost:8080

## Конфигурация

`application.yml` — только список генераторов и SSH-порт/таймауты:

```yaml
monitor:
  ssh:
    port: 22
    connect-timeout-ms: 10000
    command-timeout-ms: 15000
  generators:
    - name: generator-01
      host: 10.0.0.1
```

Логин/пароль **не хранятся в конфиге** — вводятся на странице входа и держатся в сессии.
